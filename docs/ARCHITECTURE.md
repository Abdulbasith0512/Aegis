# AegisAI System Architecture

This document describes the high-level system components and supporting services that comprise the AegisAI Banking AI Governance Operating System.

---

## 1. High-Level Components Flow

Below is the execution flow of a client transaction request as it transitions through the AegisAI governance subsystems:

```
    [ Customer ]
         │
         ▼
   [ API Gateway ]
         │
         ▼
  [ Authentication ]
         │
         ▼
  [ AI Orchestrator ]
         │
         ├──► [ Fraud Agent ]
         ├──► [ AML Agent ]
         ├──► [ KYC Agent ]
         ├──► [ Behavior Agent ]
         ├──► [ Device Agent ]
         └──► [ Compliance Agent ]
         │
         ▼
  [ Decision Engine ]
         │
         ▼
[ Trust Score Engine ]
         │
         ▼
   [ Policy Engine ]
         │
         ▼
  [ Human Review ] (Conditional Escalation)
         │
         ▼
  [ Audit Ledger ]
         │
         ▼
    [ Dashboard ]
```

## 2. Component Details

### 2.1 Customer / Client Interface
The entrypoint represents public client applications (mobile banking apps, core ledger interfaces, or external APIs) dispatching financial requests.

### 2.2 API Gateway
An asynchronous FastAPI reverse proxy that intercepts all incoming API requests. It standardizes headers, executes early rate limiting, and routes requests to the Control Plane.

### 2.3 Authentication
Validates client and tenant API keys, checks OAuth2 scopes, and ensures request signatures are mathematically verifiable prior to payload execution.

### 2.4 AI Orchestrator
The state-machine controller that coordinates agent evaluations. It aggregates background variables, triggers dependencies, and manages the execution flow of the domain agents.

### 2.5 Domain Agents Layer
* **Fraud Agent**: Evaluates transaction metadata to classify immediate transaction fraud risk.
* **AML Agent**: Scans user graphs to identify structuring and placement indicators.
* **KYC Agent**: Processes customer documents, performs OCR check, and matches identity registries.
* **Behavior Agent**: Monitors telemetry activity against the customer's historical footprint clusters.
* **Device Agent**: Evaluates emulator signatures, network profiles, and IP blacklists.
* **Compliance Agent**: Ensures the proposed transaction matches strict legal check boundaries.

### 2.6 Decision Engine
Consolidates the classification outputs, anomaly scores, and flags generated by the Domain Agents into a unified logical evaluation.

### 2.7 Trust Score Engine
Calculates the dynamic Trust Score (0-100) using a weighted risk formula combining model certainty, data drift metrics, device profile security, and rule exceptions.

### 2.8 Policy Engine
A deterministic rule runner that checks the transaction outputs against business constraints and compliance guidelines. It triggers alarms if thresholds are breached.

### 2.9 Human Review
A portal interface that prompts human-in-the-loop (HITL) approval when the Trust Score falls below threshold values or when critical policies are flagged.

### 2.10 Audit Ledger
An immutable database log (PostgreSQL with pgvector for semantic search) that records transaction traces, audit results, and explains verdicts securely.

### 2.11 Dashboard
A Next.js admin portal that provides administrators with real-time views of active agents, trace records, alert statuses, and chaos testing scenarios.

## 3. Supporting Services

AegisAI utilizes specialized databases, vector indexes, caching layers, and telemetry suites:

- **PostgreSQL**: Stores relational tables, user configurations, policy rules, and historical audit trail records.
- **Neo4j**: Graph database mapping user accounts, organizations, and transfer edges to identify AML structures.
- **Qdrant**: Vector database hosting semantic configurations and malicious injection prompt templates.
- **Redis**: Low-latency key-value store for session caching, rate-limit counting, and WebSocket subscription pools.
- **Prometheus**: Scrapes operational parameters (request latency, agent latency, HTTP status codes).
- **Grafana**: Visualizes metrics dashboards, system status maps, and policy statistics.
- **MLflow**: Tracks model parameter histories, evaluation parameters, and model drift datasets.
